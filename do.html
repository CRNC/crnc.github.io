What are you waiting for? Just Do it -- Embrace reproducibility from the get-go

Our computational environments and infrastructure grow more powerful, more heterogeneous, and more complicated. Reproducibility can be improved through practices and tools which establish efficient management of data, analyses scripts, and facilitation of analysis execution in a fashion that they could later be inspected, compared, validated, adjusted or expanded, and re-executed. To have the most of effect such solutions should consider all stages of the scientific process, from the experiment planing and data collection, to publication of the findings. We aim to

ReproIn

Collect and prepare neuroimaging data in an efficient and automated way to immediately benefit from the established by the community standard -- Brain Imaging Data Structure (BIDS) -- and Distributed data distribution and management system DataLad. To achieve that we

  • establish a specification and procedures for MRI scanning sessions to allow for flexible, automated, and non-ambiguous conversion of all acquired MR data from DICOM files to BIDS dataset(s)
  • contribute to the development of the Heuristic DICOM Converter (HeuDiConv) utility to perform the conversion and version control of all data using DataLad
  • assure robust operation, wide dissemination, and detailed documentation for the project to ease adoption

Having all collected data automatically converted to BIDS, while retaining all original DICOMs as well, makes it possible to

  • validate correct data collection by using the BIDS-validator and detect possible problems with the design or acquisition in early (pilot) stages of the project
  • avoid many common mistakes stemming from manual data conversion and curation, thus making input data itself validated and reproducible
  • automatically reconvert all data in case if relevant defects get detected in the conversion backend utility, possibly again to reconfirm reproducibility of the data itself
  • facilitate data sharing (private or public) and publication (data descriptor papers) since all data is immediate almost completely ready for sharing in a standardized files layout and annotation

NICEMAN

ReproNim is developing NICEMAN to help researchers manage, track and share information in a comprehensive format. Brainverse is an open-source, cross-platform desktop application to enable researchers add to reproducible practices into every project.

  • An electronic laboratory notebook built as a cross platform desktop application
  • Enables users to plan experiments, collect, analyze and reuse data, and collaborate
  • Adds semantic annotation to data with relevant metadata based on NeuroImaging data Model (NIDM) making experimental neuroimaging study more reproducible, and making data FAIR
  • Intercept the research workflow at planning stage, curating (raw, processed, results) data at the source and helping users to annotate every step of the process.

We are looking for beta users of the software and JavaScript developers to extend BrainVerse.
